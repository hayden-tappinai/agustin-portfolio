Eight coded interactions, full stop. Drawn / character motion (the soccer-ball fly-through, any hand-drawn figure) is not coded here — it's a Higgsfield asset dropped into a labeled slot (see below). A flat b/w ball glyph may sit as a static motif, but it never animates.

Drawn motion → Higgsfield slot (§8)
HIGGSFIELD ASSET — added later soccer-ball fly-through · b/w hand-drawn · transparent WebM · ~480×480 · loops sits near the cursor / over a lifting country on the globe · trigger: hover a country

Why a slot, not CSS: Claude-coded character animation looks bad. Drawn motion is generated in Higgsfield (start frame + end frame on green → animated → background stripped) and dropped in later. The design system reserves the space and specs the asset; it never fakes it.

// the signature micro-interaction

// on .torn — stub detaches at the seam
.stub.torn {
  transform: translateY(86px) rotate(26deg);
  opacity: 0;            // falls + spins + fades
  transition: 520ms var(--ease-thunk);
}

WHY the whole system runs on the boarding-pass motif, so its tear is the showcase. Framer Motion: AnimatePresence on the stub, spring {stiffness:300,damping:25}, exit on y / rotate / opacity.

Paper glide — corner peel, polaroid develop

--ease-draw

cubic-bezier(.65,0,.35,1)
Route draw / count-up — dotted flight path, stat tally

Read it like a race: all three runners launch together and travel the same distance in 2.8s, so the curve's character is visible side-by-side — thunk overshoots past the line and snaps back, paper glides in clean, draw eases out hard like a count-up landing on its final number. These are the three featured easing tokens (the old ball-roll curve is gone — ball motion is a Higgsfield asset, not coded). Tokens live in :root so React reads the identical bezier.
